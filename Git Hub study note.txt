基本操作
1 初始化仓库
$ git init

2 查看仓库状态
$ git status

3 向暂存区添加文件
$ git add .

4 从暂存区向仓库添加
$ git commit -m "message"

5 查看提交日志
$ git log --pretty=short
$ git log -p

6 查看工作树和暂存区的区别
$ git diff

good habit:
在commit之前先执行 git diff HEAD 查看本次提交与最新提交的区别
然后直接commit
在 git log 看提交是否成功

+ 表示新添加
- 表示删除

分支操作
master分支是默认分支

git branch
* 分支号前面有*表示当前分支

git checkout -b
以当前分支为基础创建新分支
// git checkout -b 分支名

实际这个命令相对于连续执行两个命令
git branch 分支名         // 创建分支
git checkout 分支名       // 切换分支

git checkout master      // 切换到master分支上

git checkout -           // 切换到上一个分支

2 合并分支

git checkout master
git merge --no-ff 要合并的分支名

查看分支图表
git log --graph

添加远程仓库

git remote add 远程仓库标识符 远程仓库地址

推送至远程仓库
将当前分支下本地仓库的内容推送到远程仓库
git push

git push -u origin master
-u 可以在推送的同事，将origin仓库的master分支设置为本地仓库当前分支的upstream
将来运行git pull命令从远程仓库获取内容时，本地仓库的这个分支就可以直接从
origin的master分支获取内容。

推送至master以外分支
1 首先在本地创建一个分支
  git checkout -b featureName

2 将本地分支推送远程仓库
  git push -u origin featureName


从远程仓库获取
git clone
ps:不能与之前操作的仓库同一目录下


git clone后默认是master同时会自动将仓库标识符设为origin

git branch -a 查看所有分支

git checkout -b feature-D origin/feature-D

git pull 获取最新的远程仓库分支
